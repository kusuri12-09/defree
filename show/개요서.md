Slide 1. 타이틀 (Impact)
[배달 앱 대신 냉장고를 켜게 만드는 힘, defree]

슬로건: 필요한 만큼만(degree), 죄책감에서 자유롭게(free)

디자인 가이드: 화면 중앙에 defree 로고를 크게 배치하고, 주변을 '스캔', '알림', '레시피' 텍스트 박스가 원형으로 감싸는 구조로 배치하세요.

Slide 2. 문제 정의 (Problem)
[750만 1인 가구의 식재료 잔혹사: "왜 결국 버리게 될까?"]
| 단계 | 현상 | 결과 |
| :--- | :--- | :--- |
| 과잉 구매 | 마트의 대용량 규격 | 식재료 과부하 |
| 망각/방치 | 냉장고 속 재료 파악 실패 | 유통기한 경과 |
| 번거로움 | 수동 입력의 귀찮음 | 관리 포기 및 배달 주문 |

디자인 가이드: 역삼각형 모양의 깔끔한 표를 사용하여 문제가 심화되는 과정을 시각화하세요.

Slide 3. 해결책 (Solution)
[1인 가구의 라이프사이클에 맞춘 'Zero-Entry' 관리]

기존 방식: 마트 장보기 → 수동 입력(실패) → 유통기한 망각 → 폐기

defree 방식: 영수증 스캔 → 자동 재고 등록 → 맞춤 알림 → 1인분 소진

디자인 가이드: 슬라이드를 좌우로 나누어 **빨간색(기존)**과 파란색(defree) 대비를 주어 강조하세요.

Slide 4. 서비스 원칙 (UX Principle)
[1인 가구만을 위한 4가지 설계 원칙]

1인분 레시피 고정: 모든 추천 분량을 1인용으로 자동 환산.

소포장 우선 추천: 장바구니 생성 시 낱개 상품 최상단 노출.

조기 알림 시스템: 1인 소비 속도를 고려한 D-3 선제적 알림.

소비 가능량 경고: "혼자 다 드시려면 3일간 3끼를 다 요리해야 해요."

Slide 5. 핵심 기술 ①: AI 재고 자동화
[난해한 상품명을 깨끗한 데이터로, AI 정규화 파이프라인]

Process: 영수증 촬영 → Clova OCR (텍스트 추출) → GPT-4o-mini (의미 분석) → 표준 DB (재고 등록)

Slide 6. 핵심 기능 ②: 연쇄 소비 레시피 엔진
[버려지는 식재료 0%를 위한 'Chain-Recommendation']

Loop 구조: 유통기한 알림 발송 → 보유 재료 기반 레시피 추천 → 조리 완료 시 재고 자동 차감 → 남은 재료로 다음 끼니 제안.

포인트: "배달비 3,500원을 아낄 수 있는 레시피"라는 실질적 경제 보상 제시.
