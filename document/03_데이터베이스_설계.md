# defree — 데이터베이스 설계

---

## 개요

- DBMS: **PostgreSQL 16**
- 컨테이너: Docker (EC2 위에서 직접 운영)
- 문자셋: `UTF-8`
- 타임존: `Asia/Seoul` (모든 `TIMESTAMPTZ` 컬럼은 UTC 저장, 앱 레이어에서 KST 변환)
- 네이밍: 테이블·컬럼 모두 `snake_case`

---

## ERD 관계 요약

```
users
 ├── ingredient_categories (글로벌 공유, users와 무관)
 ├── ingredients          (user_id FK)
 ├── receipts             (user_id FK)
 │    └── receipt_items   (receipt_id FK → ingredients FK)
 ├── notification_settings (user_id 1:1)
 ├── notification_logs    (user_id FK → ingredients FK)
 ├── shopping_lists       (user_id FK, 활성 1개)
 │    └── shopping_list_items (shopping_list_id FK)
 ├── recipes              (글로벌 공유, 외부 API 캐시)
 │    └── recipe_ingredients (recipe_id FK)
 └── cooking_logs         (user_id FK → recipes FK)
      └── cooking_log_items (cooking_log_id FK → ingredients FK)
```

## 1. users

사용자 계정 정보. OAuth 소셜 로그인 전용으로 password는 관리하지 않는다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 사용자 식별자 |
| `email` | `VARCHAR(320)` | `NOT NULL, UNIQUE` | 소셜 계정 이메일 |
| `name` | `VARCHAR(100)` | `NOT NULL` | 표시 이름 |
| `provider` | `VARCHAR(20)` | `NOT NULL, CHECK (provider IN ('google','kakao'))` | OAuth 제공자 |
| `provider_id` | `VARCHAR(255)` | `NOT NULL` | 제공자 측 고유 ID |
| `refresh_token_hash` | `VARCHAR(255)` | `NULL` | Refresh Token 해시 (bcrypt) |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 가입일시 |
| `updated_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 최종 수정일시 |
| `deleted_at` | `TIMESTAMPTZ` | `NULL` | 소프트 삭제일시 |

**제약 조건**
```sql
CONSTRAINT uq_users_provider UNIQUE (provider, provider_id)
```

**인덱스**
```sql
CREATE UNIQUE INDEX idx_users_email        ON users (email);
CREATE UNIQUE INDEX idx_users_provider     ON users (provider, provider_id);
CREATE INDEX        idx_users_deleted_at   ON users (deleted_at) WHERE deleted_at IS NULL;
```

## 2. ingredient_categories

식재료 카테고리 및 1인 가구 기준 유통기한·알림 선행일 기본값. 서비스 운영자가 관리하는 글로벌 테이블이다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `SMALLINT` | `PK, GENERATED ALWAYS AS IDENTITY` | 카테고리 식별자 |
| `name` | `VARCHAR(50)` | `NOT NULL, UNIQUE` | 카테고리명 (예: 잎채소, 육류) |
| `icon_emoji` | `VARCHAR(10)` | `NOT NULL` | UI 표시용 이모지 (예: 🥬) |
| `default_expiry_days` | `SMALLINT` | `NOT NULL, CHECK (default_expiry_days > 0)` | 구매일 기준 권장 소비 기한(일) |
| `notification_lead_days` | `SMALLINT` | `NOT NULL, CHECK (notification_lead_days > 0)` | 유통기한 만료 N일 전 알림 발송 (1인 기준) |
| `can_freeze` | `BOOLEAN` | `NOT NULL, DEFAULT false` | 냉동 보관 가능 여부 |
| `frozen_expiry_days` | `SMALLINT` | `NULL` | 냉동 시 추가 보관 가능 기간(일) |
| `single_serving_unit` | `VARCHAR(20)` | `NOT NULL` | 1인 1회 소비 단위 (예: 80g, 1개) |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 생성일시 |

**시드 데이터 예시**

| name | default_expiry_days | notification_lead_days | can_freeze |
|------|--------------------|-----------------------|------------|
| 잎채소 | 3 | 2 | false |
| 두부·콩류 | 5 | 3 | true |
| 육류 | 3 | 2 | true |
| 근채류 | 10 | 4 | false |
| 계란 | 14 | 4 | false |
| 유제품 | 7 | 3 | false |
| 가공식품 | 90 | 7 | false |
| 소스·양념 | 90 | 14 | false |

**인덱스**
```sql
CREATE UNIQUE INDEX idx_ingredient_categories_name ON ingredient_categories (name);
```

## 3. ingredients

사용자별 냉장고 재고 목록. 1인 가구 소비량 경고 계산의 핵심 테이블이다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 재고 식별자 |
| `user_id` | `UUID` | `NOT NULL, FK → users(id)` | 소유 사용자 |
| `category_id` | `SMALLINT` | `NOT NULL, FK → ingredient_categories(id)` | 카테고리 |
| `name` | `VARCHAR(100)` | `NOT NULL` | 품목명 (정규화된 이름) |
| `quantity` | `NUMERIC(8,2)` | `NOT NULL, CHECK (quantity >= 0)` | 재고 수량 |
| `unit` | `VARCHAR(20)` | `NOT NULL` | 단위 (개, g, ml, 봉 등) |
| `purchase_date` | `DATE` | `NOT NULL` | 구매일 |
| `expiry_date` | `DATE` | `NOT NULL` | 소비 기한 |
| `is_frozen` | `BOOLEAN` | `NOT NULL, DEFAULT false` | 냉동 보관 여부 |
| `frozen_at` | `DATE` | `NULL` | 냉동 전환일 |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'active', CHECK (status IN ('active','consumed','expired','discarded'))` | 재고 상태 |
| `source` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'manual', CHECK (source IN ('ocr','manual','digital'))` | 등록 경로 |
| `receipt_item_id` | `UUID` | `NULL, FK → receipt_items(id)` | OCR 등록 시 연결된 영수증 항목 |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 등록일시 |
| `updated_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 최종 수정일시 |

**제약 조건**
```sql
CONSTRAINT chk_ingredients_frozen
  CHECK (is_frozen = false OR (is_frozen = true AND frozen_at IS NOT NULL))

CONSTRAINT chk_ingredients_expiry
  CHECK (expiry_date >= purchase_date)
```

**인덱스**
```sql
-- 홈 대시보드: 유저의 활성 재고를 유통기한 순으로 조회
CREATE INDEX idx_ingredients_user_status_expiry
  ON ingredients (user_id, status, expiry_date)
  WHERE status = 'active';

-- 스케줄러: 전체 D-day 계산 (매일 오전 9시 배치)
CREATE INDEX idx_ingredients_expiry_status
  ON ingredients (expiry_date, status)
  WHERE status = 'active';

-- 카테고리별 재고 필터
CREATE INDEX idx_ingredients_user_category
  ON ingredients (user_id, category_id)
  WHERE status = 'active';
```

## 4. receipts

영수증 스캔 이력. OCR 처리 상태를 추적한다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 영수증 식별자 |
| `user_id` | `UUID` | `NOT NULL, FK → users(id)` | 스캔한 사용자 |
| `image_url` | `VARCHAR(500)` | `NOT NULL` | S3 presigned URL 또는 저장 경로 |
| `ocr_raw_text` | `TEXT` | `NULL` | Clova OCR 원본 추출 텍스트 |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'pending', CHECK (status IN ('pending','processing','completed','failed'))` | OCR 처리 상태 |
| `error_message` | `TEXT` | `NULL` | 실패 시 오류 메시지 |
| `scanned_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 스캔 요청 일시 |
| `completed_at` | `TIMESTAMPTZ` | `NULL` | OCR 완료 일시 |

**인덱스**
```sql
-- 사용자별 스캔 이력 조회
CREATE INDEX idx_receipts_user_scanned
  ON receipts (user_id, scanned_at DESC);

-- 스케줄러: 미처리 건 재시도
CREATE INDEX idx_receipts_status_pending
  ON receipts (status, scanned_at)
  WHERE status IN ('pending', 'processing');
```

## 5. receipt_items

영수증 내 개별 품목. OCR 원본명과 GPT 정규화명을 모두 보관해 품질 개선에 활용한다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 항목 식별자 |
| `receipt_id` | `UUID` | `NOT NULL, FK → receipts(id) ON DELETE CASCADE` | 소속 영수증 |
| `raw_name` | `VARCHAR(200)` | `NOT NULL` | OCR 원본 상품명 (예: `삼양)라120*5`) |
| `normalized_name` | `VARCHAR(100)` | `NULL` | GPT 정규화 품목명 (예: `라면`) |
| `quantity` | `NUMERIC(8,2)` | `NOT NULL, DEFAULT 1` | 수량 |
| `unit` | `VARCHAR(20)` | `NOT NULL, DEFAULT '개'` | 단위 |
| `is_confirmed` | `BOOLEAN` | `NOT NULL, DEFAULT false` | 사용자 확인 여부 (SCR-04 저장 후 true) |
| `ingredient_id` | `UUID` | `NULL, FK → ingredients(id)` | 생성된 재고 ID (확인 후 연결) |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 생성일시 |

**인덱스**
```sql
CREATE INDEX idx_receipt_items_receipt_id ON receipt_items (receipt_id);
CREATE INDEX idx_receipt_items_ingredient  ON receipt_items (ingredient_id) WHERE ingredient_id IS NOT NULL;
```

## 6. recipes

외부 레시피 API(만개의 레시피) 응답을 캐싱하는 글로벌 테이블.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 내부 레시피 식별자 |
| `external_id` | `VARCHAR(100)` | `NOT NULL, UNIQUE` | 외부 API 고유 ID |
| `source` | `VARCHAR(30)` | `NOT NULL, CHECK (source IN ('mangae','youtube','gpt'))` | 레시피 출처 |
| `title` | `VARCHAR(200)` | `NOT NULL` | 요리명 |
| `servings` | `SMALLINT` | `NOT NULL, DEFAULT 1, CHECK (servings >= 1)` | 인분 수 (defree 기본 1인분) |
| `cook_time_minutes` | `SMALLINT` | `NULL, CHECK (cook_time_minutes > 0)` | 조리 시간(분) |
| `thumbnail_url` | `VARCHAR(500)` | `NULL` | 썸네일 이미지 URL |
| `source_url` | `VARCHAR(500)` | `NULL` | 원본 레시피 URL |
| `youtube_url` | `VARCHAR(500)` | `NULL` | 유튜브 영상 URL |
| `description` | `TEXT` | `NULL` | 요리 설명 |
| `cached_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 캐시 저장 일시 |
| `expires_at` | `TIMESTAMPTZ` | `NOT NULL` | 캐시 만료 일시 (기본 7일) |

**인덱스**
```sql
CREATE UNIQUE INDEX idx_recipes_external_id ON recipes (external_id);
CREATE INDEX        idx_recipes_expires_at  ON recipes (expires_at);
```

## 7. recipe_ingredients

레시피별 필요 재료 목록. 레시피 추천 필터링 엔진의 핵심 조회 대상이다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 식별자 |
| `recipe_id` | `UUID` | `NOT NULL, FK → recipes(id) ON DELETE CASCADE` | 소속 레시피 |
| `ingredient_name` | `VARCHAR(100)` | `NOT NULL` | 재료명 (정규화 표준명 기준) |
| `quantity` | `NUMERIC(8,2)` | `NOT NULL` | 필요 수량 (1인분 기준) |
| `unit` | `VARCHAR(20)` | `NOT NULL` | 단위 |
| `is_optional` | `BOOLEAN` | `NOT NULL, DEFAULT false` | 없어도 되는 선택 재료 여부 |
| `is_seasoning` | `BOOLEAN` | `NOT NULL, DEFAULT false` | 양념·소스류 여부 (필터링 시 가중치 다름) |

**인덱스**
```sql
CREATE INDEX idx_recipe_ingredients_recipe_id ON recipe_ingredients (recipe_id);

CREATE INDEX idx_recipe_ingredients_name ON recipe_ingredients (ingredient_name);
```

## 8. notification_settings

사용자별 알림 채널 설정. users와 1:1 관계이며 row는 최초 설정 시 생성된다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 식별자 |
| `user_id` | `UUID` | `NOT NULL, UNIQUE, FK → users(id) ON DELETE CASCADE` | 사용자 |
| `web_push_enabled` | `BOOLEAN` | `NOT NULL, DEFAULT false` | FCM Web Push 사용 여부 |
| `web_push_token` | `VARCHAR(500)` | `NULL` | FCM 구독 토큰 |
| `discord_enabled` | `BOOLEAN` | `NOT NULL, DEFAULT false` | Discord Webhook 사용 여부 |
| `discord_webhook_url_enc` | `TEXT` | `NULL` | Discord Webhook URL (AES-256 암호화) |
| `slack_enabled` | `BOOLEAN` | `NOT NULL, DEFAULT false` | Slack Webhook 사용 여부 |
| `slack_webhook_url_enc` | `TEXT` | `NULL` | Slack Webhook URL (AES-256 암호화) |
| `notify_time` | `TIME` | `NOT NULL, DEFAULT '09:00:00'` | 알림 발송 시각 (KST) |
| `lead_days_override` | `SMALLINT` | `NULL, CHECK (lead_days_override > 0)` | 알림 선행일 직접 설정 (NULL이면 카테고리 기본값 사용) |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 생성일시 |
| `updated_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 최종 수정일시 |

**제약 조건**
```sql
-- Web Push 활성화 시 토큰 필수
CONSTRAINT chk_notification_web_push
  CHECK (web_push_enabled = false OR web_push_token IS NOT NULL)

-- Discord 활성화 시 URL 필수
CONSTRAINT chk_notification_discord
  CHECK (discord_enabled = false OR discord_webhook_url_enc IS NOT NULL)

-- Slack 활성화 시 URL 필수
CONSTRAINT chk_notification_slack
  CHECK (slack_enabled = false OR slack_webhook_url_enc IS NOT NULL)
```

**인덱스**
```sql
CREATE UNIQUE INDEX idx_notification_settings_user_id ON notification_settings (user_id);

-- 스케줄러: 특정 시각에 알림 발송할 사용자 조회
CREATE INDEX idx_notification_settings_notify_time
  ON notification_settings (notify_time)
  WHERE web_push_enabled = true OR discord_enabled = true OR slack_enabled = true;
```

## 9. notification_logs

발송된 알림의 이력. 중복 발송 방지 및 발송 실패 재시도에 사용한다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 식별자 |
| `user_id` | `UUID` | `NOT NULL, FK → users(id) ON DELETE CASCADE` | 수신 사용자 |
| `ingredient_id` | `UUID` | `NULL, FK → ingredients(id) ON DELETE SET NULL` | 알림 대상 재고 |
| `channel` | `VARCHAR(20)` | `NOT NULL, CHECK (channel IN ('web_push','discord','slack'))` | 발송 채널 |
| `message` | `TEXT` | `NOT NULL` | 발송 메시지 본문 |
| `status` | `VARCHAR(20)` | `NOT NULL, DEFAULT 'sent', CHECK (status IN ('sent','failed','retrying'))` | 발송 상태 |
| `error_message` | `TEXT` | `NULL` | 실패 시 오류 내용 |
| `sent_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 발송 시도 일시 |
| `retry_count` | `SMALLINT` | `NOT NULL, DEFAULT 0` | 재시도 횟수 |

**인덱스**
```sql
-- 중복 발송 방지: 같은 재료에 오늘 이미 보냈는지 확인
CREATE INDEX idx_notification_logs_user_ingredient_date
  ON notification_logs (user_id, ingredient_id, sent_at DESC);

-- 실패 건 재시도 조회
CREATE INDEX idx_notification_logs_failed
  ON notification_logs (status, sent_at)
  WHERE status IN ('failed', 'retrying');
```

## 10. shopping_lists

사용자별 장바구니. 사용자당 활성(active) 장바구니는 1개만 허용한다.

| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 장바구니 식별자 |
| `user_id` | `UUID` | `NOT NULL, FK → users(id) ON DELETE CASCADE` | 소유 사용자 |
| `is_active` | `BOOLEAN` | `NOT NULL, DEFAULT true` | 활성 여부 (완료 처리 시 false) |
| `created_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 생성일시 |
| `completed_at` | `TIMESTAMPTZ` | `NULL` | 장보기 완료 일시 |

**제약 조건**
```sql
-- 사용자당 활성 장바구니 1개 제한
CREATE UNIQUE INDEX uq_shopping_lists_active_user
  ON shopping_lists (user_id)
  WHERE is_active = true;
```

**인덱스**
```sql
CREATE INDEX idx_shopping_lists_user_active
  ON shopping_lists (user_id, is_active)
  WHERE is_active = true;
```

## 11. shopping_list_items

장바구니 내 개별 추천 품목.

| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 식별자 |
| `shopping_list_id` | `UUID` | `NOT NULL, FK → shopping_lists(id) ON DELETE CASCADE` | 소속 장바구니 |

**인덱스**
```sql
CREATE INDEX idx_shopping_list_items_list_id
  ON shopping_list_items (shopping_list_id);

CREATE INDEX idx_shopping_list_items_unpurchased
  ON shopping_list_items (shopping_list_id, is_purchased)
```

## 12. cooking_logs

레시피 조리 완료 이력. 재고 차감의 트리거이자 소비 패턴 학습의 원천 데이터다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|
| `id` | `UUID` | `PK, DEFAULT gen_random_uuid()` | 식별자 |
| `user_id` | `UUID` | `NOT NULL, FK → users(id) ON DELETE CASCADE` | 조리한 사용자 |
| `recipe_id` | `UUID` | `NULL, FK → recipes(id) ON DELETE SET NULL` | 조리한 레시피 (NULL: 레시피 없이 직접 조리) |
| `servings_cooked` | `SMALLINT` | `NOT NULL, DEFAULT 1, CHECK (servings_cooked >= 1)` | 조리 인분 수 |
| `cooked_at` | `TIMESTAMPTZ` | `NOT NULL, DEFAULT now()` | 조리 완료 일시 |

**인덱스**
```sql
-- 소비 패턴 분석: 최근 30일 조리 이력
CREATE INDEX idx_cooking_logs_user_cooked_at
  ON cooking_logs (user_id, cooked_at DESC);
```

## 13. cooking_log_items

조리 시 실제로 차감된 재고 항목. `cooking_logs`의 자식 테이블이다.

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|--------|------|-----------|------|

## 공통 설계 규칙

### UUID 생성
- PostgreSQL `pgcrypto` 확장의 `gen_random_uuid()` 사용
- 초기화: `CREATE EXTENSION IF NOT EXISTS pgcrypto;`

### 소프트 삭제
- `users` 테이블만 `deleted_at` 소프트 삭제 적용
- 나머지 테이블은 `status` 컬럼으로 상태 전환 (물리 삭제 지양)

### updated_at 자동 갱신
- 트리거로 `UPDATE` 시 자동 갱신
```sql
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = now();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 각 테이블에 적용 (예: ingredients)
CREATE TRIGGER trg_ingredients_updated_at
  BEFORE UPDATE ON ingredients
  FOR EACH ROW EXECUTE FUNCTION set_updated_at();
```

### Webhook URL 암호화
- `discord_webhook_url_enc`, `slack_webhook_url_enc`: AES-256-GCM 암호화 후 저장
- 암호화 키는 환경변수로 관리 (`WEBHOOK_ENCRYPTION_KEY`), DB에 저장 금지
- 응답 시 앱 레이어에서 복호화 후 마스킹 처리 (`https://discord.com/api/webhooks/****`)

### 파티셔닝 (향후)
- `notification_logs`: 월별 범위 파티셔닝 (`sent_at` 기준) — MAU 1,000명 초과 시 적용
- `cooking_logs`: 월별 범위 파티셔닝 (`cooked_at` 기준)
