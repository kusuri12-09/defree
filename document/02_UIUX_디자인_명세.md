# defree — UI/UX 디자인 명세서

---

## 1. 개요 (Overview)

### 프로젝트 개요

**defree**는 영수증 OCR 스캔 한 번으로 냉장고 재고를 자동 관리하고, 유통기한 임박 알림과 AI 레시피 추천을 통해 1인 가구의 식재료 낭비를 줄이는 **생활 밀착형 PWA 서비스**다.

| 항목 | 내용 |
|------|------|
| 서비스 형태 | React PWA (Progressive Web App) |
| 주요 플랫폼 | 모바일 브라우저 (Android Chrome 우선), 데스크탑 |
| 핵심 가치 | 입력 최소화 · 낭비 방지 · 결정 피로 해소 |

### 타겟 사용자

| 구분 | 프로필 |
|------|--------|
| 1차 타겟 | 20대 사회 초년생, 자취 초보, 요리 경험 적음 |
| 2차 타겟 | 식비 절감에 관심 있는 1인 가구 전반 |
| 공통 특성 | 스마트폰 중심 생활, 배달앱 자주 사용, Discord·Slack 익숙 |

#### 사용자 Pain Point

- 냉장고에 뭐가 있는지 몰라서 배달을 시킨다
- 영수증·재고를 직접 입력하기 귀찮아 앱을 이탈한다
- 유통기한이 지난 식재료를 버릴 때 죄책감과 금전적 손실을 동시에 느낀다

## 2. 기능 명세 (Functional Specifications)

### 2-1. 주요 기능 목록

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 영수증 스캔 & 재고 자동 등록 | OCR로 영수증 인식, LLM 품목 정규화 후 재고 자동 저장 | P0 |
| 냉장고 재고 조회 | 보유 식재료 목록, 유통기한, 남은 수량 확인 | P0 |
| 유통기한 임박 알림 | D-2 시점에 Discord·Slack·Web Push로 알림 발송 | P0 |
| 레시피 추천 | 보유 재료 기반 오늘 저녁 + 내일 아침 연쇄 추천 | P0 |
| 스마트 장바구니 | 소비 패턴 분석 기반 이번 주 구매 목록 자동 생성 | P1 |
| 알림 채널 설정 | Discord·Slack Webhook URL, FCM Web Push 온/오프 설정 | P1 |
| 재고 수동 입력·수정 | 직접 식재료 추가, 수량·유통기한 수정 | P1 |
| 조리 완료 처리 | 레시피 사용 후 재료 자동 차감 | P1 |

```
│  └─────────────────────┘    │
│  ┌─────────────────────┐    │

```

```
│                             │
│  ┌─────────────────────┐    │

│  │  두부        D-2     │    │
│  │  → 바로 레시피 보기  │    │
│  └─────────────────────┘    │
│                             │
│  오늘의 추천 레시피          │
│  ┌───────┐ ┌───────┐        │
│  │ 파스타 │ │ 된장국 │        │

│  └───────┘ └───────┘        │
│                             │

│  │  📷  영수증 스캔하기 │    │  ← FAB (고정)
```

**UI 요소**
- 유통기한 임박 재료 카드 (D-2 이하 강조)

- 영수증 스캔 FAB (Floating Action Button) — 화면 하단 고정
- 하단 탭바: 홈 / 냉장고 / 레시피 / 장보기 / 마이

**인터랙션**
- 임박 재료 카드 → 해당 재료 기반 SCR-08 (레시피 목록) 이동
- 레시피 카드 → SCR-09 (레시피 상세)
- FAB 탭 → SCR-03 (영수증 스캔)

### 3-2. 구현 세부사항

#### 상태 관리 구조

```
Zustand Store
├── authStore       — 사용자 인증 정보 (토큰, 프로필)
├── ingredientStore — 재고 목록 (로컬 캐시)
└── notificationStore — 알림 설정 (채널, 시각)

TanStack Query
├── useIngredients()   — GET /ingredients
├── useScanReceipt()   — POST /ocr/scan (mutation)
├── useRecipes()       — GET /recipes?ingredientIds=...
└── useShoppingList()  — GET /shopping-list
```

#### API 연동 규칙

- 모든 API 요청은 `packages/shared-types` 의 DTO 타입 공유
- 인증 헤더: `Authorization: Bearer {JWT}`
- 에러 응답 공통 포맷: `{ code, message, details }`
- Axios 인터셉터로 401 시 자동 토큰 갱신 처리

#### PWA & Web Push

```
Service Worker 등록 (앱 최초 접속)
  → Notification.requestPermission() 요청
  → FCM 구독 토큰 발급
  → POST /notifications/subscribe { token, channel: 'fcm' }
  → 서버 저장 완료

알림 수신 (백그라운드)
  → Service Worker onpush 이벤트
  → self.registration.showNotification()
  → 탭 클릭 시 딥링크 URL로 앱 포커스
```

## 5. 비기능 명세 (Non-Functional Specifications)

### 5-1. 성능 요구사항

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| 첫 화면 로딩 (FCP) | < 1.5초 | Lighthouse |
| 상호작용 가능 시간 (TTI) | < 3초 | Lighthouse |
| OCR 처리 응답 | < 5초 | 서버 로그 |
| 레시피 추천 응답 | < 1초 | API p95 |
| Lighthouse PWA 점수 | ≥ 90 | Lighthouse |

- 이미지 최적화: WebP 변환, `loading="lazy"`
- 코드 스플리팅: 라우트 단위 `React.lazy` + `Suspense`
- 레시피 결과 캐싱: TanStack Query `staleTime: 5분`

---

### 5-2. 접근성

- 기준: **WCAG 2.1 AA** 준수
- 색상 대비: 텍스트 최소 4.5:1, 대형 텍스트 3:1 이상
- 이미지 대체 텍스트: 모든 `<img>`에 `alt` 속성 필수
- 키보드 내비게이션: Tab 포커스 순서 논리적 유지, `focus-visible` 스타일 명시
- 스크린 리더: ARIA 레이블 (`aria-label`, `aria-live`, `role`) 적용
  - 유통기한 임박 알림 카드: `role="alert"`
  - 로딩 스피너: `aria-busy="true"`, `aria-label="처리 중"`

---

### 5-3. 보안 요구사항

| 항목 | 구현 방식 |
|------|-----------|
| 인증 | Google / Kakao OAuth 2.0 + JWT (Access Token 15분, Refresh Token 7일) |
| 토큰 저장 | Access Token: 메모리(Zustand), Refresh Token: HttpOnly Cookie |
| HTTPS | Let's Encrypt (MVP), 전 구간 강제 리다이렉트 |
| XSS 방지 | React 기본 이스케이프 + `dangerouslySetInnerHTML` 사용 금지 |
| CSRF | SameSite=Strict Cookie + CSRF 토큰 헤더 |
| Webhook URL | 서버 저장 시 AES-256 암호화, 응답 시 마스킹 표시 |
| 이미지 업로드 | 파일 타입·크기 서버 검증 (MIME, 10MB 이하), S3 presigned URL 사용 |
